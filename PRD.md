# 반도체 시료 생산주문관리 시스템
프로젝트 목표: 반도체 시료를 주문/생산/출고하는 간단한 시스템을 구현한다.

## 참조 필요 내용
현재 프로젝트 상위 디렉토리에 존재하는 4개의 프로젝트를 참조하여 반도체 시료 생산주문관리 시스템(SampleOrderSystem)을 구현한다.
참조 프로젝트는: ConsoleMVC, DataMonitor, DataPersistence, DummyDataGenerator
각 프로젝트는 SampleOrderSystem의 일부 기능들에 대하여 PoC를 진행한 것들이다.
본 프로젝트는 위 프로젝트들을 참조하여 요구사항에 맞게 잘 조합하여 기능을 구현하는것이 목표.
README를 참고하여 어떤 기능들이 존재하는지 확인하여 아키텍처 설계 시 반영한다.

## 데이터 저장 방식
DB 서비스를 통해 데이터를 관리하며, 인메모리 방식이 아닌 서버 형태로 동작한다.
주문ID 등 시퀀스 기반 ID는 DB 데이터를 조회하여 생성한다.

## 주문 상태 전환

```
[주문 등록] → RESERVED
RESERVED    → CONFIRMED   (승인 + 재고 충분: 즉시 재고 차감 후 배송대기 재고에 추가)
RESERVED    → PRODUCING   (승인 + 재고 부족: 생산 요청 등록)
RESERVED    → REJECTED    (거절)
PRODUCING   → CONFIRMED   (생산 완료 시 자동: 재고 차감 후 배송대기 재고에 추가)
CONFIRMED   → RELEASE     (출고 처리: 배송대기 재고 차감)
```

- `RESERVED`: 주문 등록 후 승인 대기 중
- `CONFIRMED`: 승인 완료, 출고 대기 중 (재고 차감 및 배송대기 재고에 추가된 상태)
- `PRODUCING`: 재고 부족으로 생산 요청이 등록된 상태
- `RELEASE`: 출고 완료
- `REJECTED`: 주문 거절

PRODUCING → CONFIRMED 전환은 생산 완료 시 **자동으로** 처리된다.

## 시스템 메뉴 구성
### 홈 화면
홈 화면에서 생산 명령을 직접 입력할 수 있다.
생산 명령은 `생산 명령 [+ 분]` 형태로 입력하며, 별도의 전용 UI 없이 홈에서 처리된다.

### 시료 관리
시료 관리 관련 기능은 시료 등록, 시료 조회, 시료 검색, 시료 수정, 시료 삭제 이렇게 다섯 가지가 존재한다.
#### 시료 등록
시료 속성을 입력하면 새로운 시료를 시스템에 추가한다.
#### 시료 조회
등록된 모든 시료 목록을 확인할 수 있다.
각 시료별로 수량도 표시된다.
#### 시료 검색
시료의 ID나 이름 등 시료 속성으로 특정 시료를 검색한다.
#### 시료 수정
등록된 시료의 속성(이름, 분당생산속도, 수율)을 수정한다.
시료 ID는 수정 불가하다.
#### 시료 삭제
등록된 시료를 시스템에서 삭제한다.
진행 중인 주문(RESERVED, PRODUCING, CONFIRMED)이 존재하는 시료는 삭제할 수 없다.

### 시료 주문
주문은 시료의 재고 수량과 관계없이 등록할 수 있다.
주문 등록 시 시료ID, 고객명, 주문 수량을 입력한다.
단, 시스템에 등록된 시료만 주문 가능하다.
입력 내용을 확인한 후 주문확정을 하면 주문이 RESERVED 상태로 생성되며 완료 내역을 출력한다.

### 주문 승인/거절
접수된 주문 목록(RESERVED)을 조회하고 특정 주문에 대하여 승인 혹은 거절을 하는 메뉴이다.
주문 번호를 선택 한 후 승인할지 거절할지 입력을 받는다.
#### 주문 승인
승인하기를 하면 현재 시료 재고 현황, 변경 후 재고 수량과 부족한 수량을 보여준 후 승인여부를 입력받는다.
##### 승인 시 분기 로직
- 재고가 충분한 경우: 주문을 즉시 CONFIRMED 상태로 전환하고 재고를 차감한 후 배송대기 재고에 추가한다.
- 재고가 부족한 경우: 생산 라인에 자동으로 생산 요청을 등록하고 주문을 PRODUCING 상태로 변경한다. 생산된 수량은 무조건 재고(Stock)에 반영되며, 필요 수량이 충족됐는지 확인하여 충분할 경우 즉시 재고를 차감하고 배송대기 재고에 추가한 뒤 주문을 CONFIRMED로 자동 전환한다.
#### 주문 거절
주문 거절을 하는 경우 주문은 REJECTED 상태로 전환된다.

### 모니터링
#### 주문량 확인
상태 별(RESERVED, CONFIRMED, PRODUCING, RELEASE) 주문 수를 확인 가능하며 REJECTED는 제외한다.
#### 재고량 확인
각 시료별 재고 현황을 아래 컬럼으로 표시한다.

| 시료ID | 시료명 | 재고 수량 | 배송대기 재고 | 총 재고 수량 | 주문된 수량 | 상태 |
|--------|--------|-----------|--------------|-------------|------------|------|

- **재고 수량**: Stock(일반 재고)의 현재 수량
- **배송대기 재고**: PendingShipmentStock의 현재 수량
- **총 재고 수량**: 재고 수량 + 배송대기 재고
- **주문된 수량**: 해당 시료의 PRODUCING/RESERVED/CONFIRMED 상태 주문 수량 합계
- **상태**: 고갈 / 부족 / 여유 세 가지 조건
  - 고갈: 재고가 0인 상태
  - 부족: 재고가 0은 아니지만 주문된 수량보다는 부족한 상태
  - 여유: 주문된 수량보다 재고가 많은 상태

### 생산라인
#### 생산 현황 표기
현재 생산 중인 항목(생산 스케줄 가장 앞에 있는 항목)의 상세 정보를 출력한다.
표시 항목: 스케줄 ID, 주문ID, 시료명, 목표 수량, 현재 생산량, 잔여 수량
생산 중인 항목이 없으면 "현재 생산 중인 항목이 없습니다." 메시지를 표시한다.
#### 대기 주문 확인
생산 대기 중인 전체 스케줄 목록을 FIFO 순서대로 페이징하여 출력한다.
표시 항목: 스케줄 ID, 주문ID, 시료명, 목표 수량, 현재 생산량, 잔여 수량

### 출고
CONFIRMED 상태인 주문을 조회하고 선택하여 RELEASE 상태로 전환한다.
RELEASE 되는 순간 배송대기 재고(PendingShipmentStock)에서 주문 수량만큼 차감된다.
출고 처리가 완료되면 출고 이력에 대하여 출력하고 메인 기능으로 돌아간다.

## 주요 클래스
### 재고 (Inventory 계층)
재고는 `Inventory` 기본 클래스를 상속받아 두 가지 타입으로 관리한다.

```
Inventory (abstract)
├── Stock          // 일반 재고
└── PendingShipmentStock  // 배송대기 재고
```

- **Inventory**: 시료ID, 수량을 공통 속성으로 가지며 재고 증감 기본 메서드를 정의한다.
- **Stock**: 생산 완료 시 수량이 증가하며, 주문 CONFIRMED 전환 시 수량이 차감된다.
- **PendingShipmentStock**: 주문 CONFIRMED 전환 시 수량이 증가하며, 출고(RELEASE) 처리 시 수량이 차감된다.

### 생산
생산 스케쥴을 큐를 활용하여 FIFO 관리한다.
주문이 PRODUCING 상태로 전환될 때 스케쥴이 추가되며, 생산 완료 시 스케쥴을 제거한다.
실제 생산량은 매 분마다 `시료별 분당 생산량 * 수율 * 0.9` 이다.
생산은 실시간이 아니며 홈 화면의 생산 명령을 통해 시뮬레이션된다.

#### 생산 명령
`생산 명령 [+ 분]` 형태로 홈에서 입력받으면 현재 스케쥴링된 생산을 입력받은 시간만큼 진행한다.
생산된 수량은 무조건 해당 시료의 Stock(재고)에 반영된다.
스케쥴에 필요한 수량만큼 생산됐는지 확인하여 충분할 경우:
- 해당 스케쥴을 제거하고 주문을 CONFIRMED로 자동 전환한다.
- Stock에서 주문 수량만큼 차감하고 PendingShipmentStock에 추가한다.

실제 생산량과 주문수량을 바탕으로 불필요한 반복을 제거하도록 최적화 수행한다.
(분당 생산량 1, 주문수량 1000, 생산시간 1000일 경우 1000번 반복하지 않고 한번에 처리)

### 주문
시스템에 등록된 시료만 주문 가능하며, 재고 수량과 관계없이 주문 등록이 가능하다.
속성은 주문ID, 시료ID, 고객명, 주문 수량 네 가지이다.
주문ID는 `YYYYMMDD_[날짜별 숫자 시퀀스]` 형태로 자동 생성된다. DB 데이터를 조회하여 생성한다.
주문ID는 PK이며 생성 이후 변경 불가하다.

### 시료
시료(Sample)는 시스템의 가장 기본이 되는 단위로 각 시료는 고유한 이름과 속성을 가진다.
속성은 시료ID, 이름, 분당생산속도, 수율 이렇게 네 가지이다.
시료ID는 숫자 시퀀스로 자동 생성되며 PK이므로 생성 이후 변경 불가하다.

## 추가 요구사항
시료/주문/생산 화면에서 데이터 조회 시 페이징 기법을 적용하여 한 화면에 너무 많은 데이터가 조회되지 않도록 한다.
페이지당 표시 건수는 상수(또는 설정 변수)로 변수화하여 쉽게 변경할 수 있도록 구현한다.
실제 화면 구현 후 적절한 값으로 조정할 예정이다.
